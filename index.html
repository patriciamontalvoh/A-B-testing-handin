Overview

During this assignment, an A/B test was conducted to analyze the user interaction with two different versions of an interface. By asking users to perform a simple task in both versions of the interface, the results were analyzed and conclusions were drawn on whether the new design choices of the interface reasonably impacted the user's interaction.

Part 1: Data Collection

• Interface analyzed: Medical center interface, appointment section.

• Task performed: Appointment scheduling with Adam Ng, MD at Morristown Medical Center on April 23, 2024.

• Number of interactions: 21 per version.

• Measurements recorded:

o Uid: Unique identifier for each user.

o Version: Version (A or B) of the website that the user interacted with.

o timestamp_load: The time (in milliseconds) elapsed since the epoch.

o time_on_page: Total time (in milliseconds) that the user spent on the page.

o time_to_first_click: The time (in milliseconds) it took for the user to execute his/her first click.

o mouse_move_distance: The total distance (in pixels) of the user's mouse movements.

o num_clicks: The number of times the user clicked on the screen.

o did_misclick: A boolean flag indicating if the user pushed a button external to the task.

o did_succeed: A boolean flag indicating if the user successfully completed the task.

User interface:

Version A: Original interface

Data collected (CSV file)

Version B: Retouched interface

Changes made in version B:

o Increased contrast between text, background, and buttons.

o Made hierarchy more notable to emphasize the most important information.

o Changed the date format.

o Rearranged of buttons, using a different style for scheduling an appointment and for viewing calendar (emphasizing the scheduling appointment button).

Goals:

o To provide a more structured and efficient user interface.

o To provide a better clarity, and therefore reduce errors and enhance productivity.

Part 2: Analysis

To assess both versions of the interface, 3 of the metrics collected were selected for its analysis. A statistical test was carried out for each of those metrics to determine whether the difference between the interface’s version A and B was statistically significant with regard to the user interaction.

The information and results of the analysis can be found underneath:

Metrics analyzed:

1. Misclick rate: Frequency with which users click something else on the page before finding the correct button for the task.

2. Time on page: Time spent on the webpage for each user group.

3. Time to first click: Time spent on the webpage before the first click was made.

1. Misclick rate

Hypotheses:

• Null Hypothesis: There is no difference in the misclick rate between version A and version B of the webpage.

• Alternative Hypothesis: Version B has a lower misclick rate compared to version A due to the UI change.

• Rationale: In the version B of the interface, the content follows a more organized structure to help the user navigate the interface efficiently and to easily understand the correct place to click. Therefore, the number of misclicks is expected to decrease.

Analysis:

• Test used: Chi-Squared Test

* Why this test? Since we are analyzing categorical data (true or false) a Chi-squared statistic is the most suitable statistical analysis to study the different outputs of data.

• Observations:

o The comparison between “true” and “false” results between version A and B shows that version B has less misclick rates than version A.

o The degrees of freedom are equal to 1 because we are analyzing a 2x2 matrix.

o The expected values show the same rates for both versions, therefore the expected distribution of encountering a misclick is the same for both versions.

o Chi-Squared Value (χ²) suggests a slight difference between the observed and expected values.

o The p-value is approximately 0.26, which is significantly higher than the standard alpha threshold of 0.05.

Interpretation:

With a p-value of 0.26, there is no significant evidence on a difference between the misclick rate of version A and version B. Therefore, we fail to reject the null hypothesis.

As a result, this analysis supports the conclusion that the modifications of version B do not affect the misclick rates compared to version A.

2. Time on page

Hypotheses:

• Null Hypothesis: Users spend the same time on the interface regardless of the changes in the interface between version A and version B.

• Alternative Hypothesis: Users spend less time on version B than on version A because of the UI modification.

• Rationale: In the version B of the UI, the content is better organized and more intuitive; therefore, users can perform tasks faster, and need less time in the interface.

Analysis:

• Test used: One-tailed t

* Why this test? Since we are analyzing two independent data samples, and we are interested in understanding the different result in just one direction (a time decrease to on the page from version B to version A) a one-tailed t test is the most suitable statistical analysis to study the different outputs of data.

• Observations:

o The variances of both versions also show a considerable difference (x20+), suggesting more diverse results in the samples from version A than version B.

o Given the notable difference between the variance of both versions, the degrees of freedom, are adjusted accordingly to adjust the shape of the T distribution to broader tails.

o The negative T-score reflects the higher values obtained testing version A in comparison to version B. Additionally, the high score suggests a high difference between the values analyzed.

o The p-value is approximately 0.000045, which is significantly lower than the standard alpha threshold of 0.05.

Interpretation:

With a p-value of 0.000045 the statistical analysis indicates a significant difference between the average in time on page in version B with comparison to version A. Therefore, we reject the null hypothesis for this metric.

As a result, this analysis supports the conclusion that version B leads to a decreased time on page compared to version A.

3. Time to first click:

Hypotheses:

• Null Hypothesis: The time spent on the webpage before the first click does not change between version A and version B.

• Alternative Hypothesis: The time spent on the webpage before the first click is different for version B than for version A.

• Rationale: Given the restructure of version B, users might a different amount of time to understand the structure of the webpage and start performing their task.

Analysis:

• Test used: Two-tailed t

* Why this test? Since we are analyzing two independent data samples, and we are interested in understanding the different results in either direction (a time increase or decrease to the first click from version B to version A), a Two-tailed t test is the most suitable statistical analysis to study the different outputs of data.

Conclusion

The A/B test performed leads to the conclusion that the UI modifications of version B do not significantly affect the misclick rate; however, they seem to be effective in reducing both the time spent on the page and the time to the first click.

These results suggest that version B provides a more efficient and user-friendly interface, which aligns with the goals of providing a more structured UI to enhance efficiency and clarity for the user.

To further confirm this hypothesis a larger scale test could be conducted.
